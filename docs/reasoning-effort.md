# 추론 수준(Reasoning Effort)  
작업의 복잡도와 속도, 비용(토큰 소모량)에 따라 선택해야 합니다. 일반적인 코딩 업무에는 Medium을 기본으로 쓰고, 복잡한 문제 해결에만 High 이상을 쓰는 것이 가장 효율적입니다.   

## 1. Medium (기본 추천)
단순 수정이나 규칙이 명확한 작업에 적합합니다.

* 추천 작업: 단순 버그 수정, 코드 리뷰, 기존 함수 리팩토링, API 연동 코드 작성.
* 장점: 답변 속도가 빠르고 토큰(비용)을 적게 소모합니다. 불필요하게 복잡한 코드를 만들지 않습니다.

## 2. High (복잡한 로직)
여러 모듈을 고려하거나 깊은 로직 설계가 필요할 때 사용합니다.

* 추천 작업: 알고리즘 최적화, 다중 조건문 처리, 새로운 기능의 프로토타입 설계.
* 장점: 추론 능력이 높아져 예외 상황(Edge Case)을 더 잘 잡아냅니다.

## 3. xHigh (최고 난이도)
처음부터 시스템 구조를 짜거나 디버깅이 완전히 막혔을 때만 사용합니다.

* 추천 작업: 대규모 아키텍처 설계, 원인 불명의 복잡한 시스템 에러 디버깅.
* 단점: 답변을 출력할 때까지 "생각하는 시간"이 오래 걸리며, 토큰 소모가 매우 큽니다.


## 1. 세션 중에 실시간으로 변경하기 (슬래시 명령어)
터미널에서 대화형(Interactive) TUI 세션을 실행 중일 때 가장 많이 쓰는 방법입니다. 

   1. Codex 대화창에 /model을 입력하고 엔터를 누릅니다.
   2. 화면에 팝업창이 뜨면 방향키(위/아래)로 움직여 원하는 모델을 선택합니다.
   3. 이어서 나오는 Effort(추론 수준) 선택 화면에서 medium, high, xhigh 중 하나를 고릅니다.
   4. 변경 후 /status를 입력하여 현재 적용된 모델과 추론 레벨을 확인할 수 있습니다. [2, 3, 4] 

  
## 2. 터미널 명령어로 일회성 실행하기 (CLI 플래그)
스크립트를 짜거나, TUI 창을 켜지 않고 특정 추론 레벨로 단발성 작업을 시킬 때 유용합니다. --config 또는 -c 플래그를 사용하여 설정을 주입합니다. 

* High 모드로 구조 분석 지시 예시:

codex --config model_reasoning_effort='"high"' "이 모듈의 구조를 분석해라"

(주의: 윈도우/리눅스 환경에 따라 따옴표 표현식 '"high"' 형태를 확인해 주세요.) 
  
  
## 3. 영구적으로 기본값 고정하기 (config.toml 수정)
매번 설정을 바꾸기 번거롭다면 사용자의 홈 디렉터리에 있는 설정 파일을 직접 수정하여 기본값을 고정할 수 있습니다. 

   1. 전역 설정 파일(~/.codex/config.toml)을 엽니다.  
   2. 파일 내에 model_reasoning_effort 옵션을 원하는 값으로 추가하거나 수정합니다.  
   ```
   model = "gpt-5.5"
   model_reasoning_effort = "medium"  # 기본값으로 고정 (low, medium, high, xhigh 가능)
   ```
